#Generally you can pip install protobuf
#then install grpcio and the versions should be compatible
#However, you should check the runtime version mentioned in 
#codegen'ed python files. Clients should be using protobuf
#of the same or greater version otherwise they will receive
#runtime errors. 

# Pinned to the versions baked into committed *_pb2.py / *_pb2_grpc.py bytes:
#   protobuf==6.31.1       → "Protobuf Python Version: 6.31.1" header
#   grpcio-tools==1.76.0   → GRPC_GENERATED_VERSION = '1.76.0' header
# An unpinned pip install drifts these strings (every fresh resolve picks up
# the current latest), producing 80+ spurious _pb2_grpc.py header diffs on
# every regen. See FinTekkers/second-brain#228 for the full diagnosis.
# Bump deliberately, not by drift.
protobuf==6.31.1 #The protobuf libraries
grpcio  #gRPC runtime — version doesn't affect generated bytes (read at
        # runtime), so left unpinned to track active wheel availability.
grpcio-tools==1.76.0  #protoc + grpc plugin for Python codegen (used by compile.sh)
